# Luna RealTime Sound System 8

## 개요
Luna RealTime Sound System (RTSS) 8.0은 실시간 오디오 처리와 정밀 동기화 기술을 구현하기 위한 커스텀 오디오 엔진입니다.

7 버전이 로컬 환경에서의 안정적인 실시간 처리에 집중했다면, 8 버전은 클라우드 연동 환경에서의 지연 시간 보정 및 네트워크 가변 상황에 대응하는 적응형 동기화 로직을 핵심으로 합니다. 현재 개발 중인 시스템은 오디오 출력 상태를 실시간으로 추적하여 가사와 오디오 간의 오차를 최소화하는 것을 목표로 합니다.

---

## RealTime Sound System의 정의
일반적인 재생 방식이 고정된 버퍼와 타이머에 의존하는 것과 달리, Luna RTSS는 특징이 많스비다
아래 글을 참조해보세요.

* **PCM 데이터 흐름 추적:** 사운드 장치로 전달되는 실제 PCM(Pulse Code Modulation) 데이터의 진행 상태를 직접 감시합니다.
* **상태 기반 동기화:** 단순 시간 계산이 아닌, 실제 출력 중인 오디오 위치를 기준으로 가사 및 시각화 데이터를 실시간 렌더링합니다.
* **저지연 설계:** 오디오 데이터 처리 계층의 병목을 제거하여 입력과 출력 사이의 반응성을 극대화합니다.

## 주요 구현 사례

### 1. PCMPlayer Cloud 8
Luna RTSS 8.0 기술이 적용된 실시간 노래방 가사 동기화 시스템입니다.
* **클라우드 동기화:** 내부 서버에서 제공되는 가사 데이터를 실시간 오디오 흐름에 맞춰 즉각적으로 매핑합니다.
* **저전력 렌더링:** 8.0 엔진의 향상된 추적 성능을 통해 반주 재생 시에도 가사 밀림 현상을 방지합니다.

### 2. TuneDesk 버전 8
디스코드 음성 채널 환경에 최적화된 RTSS 8.0의 네트워크 확장 버전입니다.
* **스트리밍 최적화:** 네트워크 지연이 발생하는 환경에서도 오디오 끊김을 최소화하는 적응형 버퍼링을 적용했습니다.
* **음질 유지:** 플랫폼 특성에 맞춘 리샘플링 최적화를 통해 원본 PCM 데이터의 손실을 최소화합니다.
* **참조:** [TuneDesk GitHub Repository](https://github.com/exaqueGithb/BotDiscordLuna-RSS)

## 기술적 특성
* **가변 버퍼 시스템**: 시스템 부하 및 네트워크 상태에 따라 버퍼 크기를 동적으로 조절하여 재생 안정성을 확보합니다.
* **모듈화 구조:** 핵심 오디오 엔진과 플랫폼별 출력 계층이 분리되어 있어 다양한 환경(데스크톱, 봇 등)에 이식이 용이합니다.
* **즉각적 상태 복구:** 재생 중 중단이나 지연 발생 시, 오디오 스트림이 복구되는 즉시 현재 시점의 동기화 데이터를 재계산하여 일치시킵니다.

## 7.0 대비 8.0 주요 변경 사항

| 구분 | Luna RTSS 7.0 | Luna RTSS 8.0 |
| :--- | :--- | :--- |
| **주요 구조** | 로컬 시스템 단일 처리 | 하이브리드 클라우드 스트리밍 대응 |
| **동기화 로직** | 고정 버퍼 상태 추적 | 적응형 가변 버퍼 및 지터(Jitter) 보정 |
| **정밀도** | 밀리초(ms) 단위 안정화 | 마이크로초 단위 데이터 흐름 분석 |
| **네트워크 대응** | 기본적인 지연 처리 | 패킷 손실 및 지연 변동에 대한 동적 복구 |
| **구현 사례** | PCMPlayer 7.0 | PCMPlayer Cloud 8 / TuneDesk 8.0 |
